## Project Overview

This is a **monorepo** containing two related projects:

1. **Flutter Mobile App** (`/`) - "诗歌 with Chords" for displaying hymnals with chords inlined. The app is primarily targeted at iOS devices, with Android support.

2. **Python Crawler** (`/crawler`) - A web scraper that fetches hymns from hymnal.net and generates the JSON data files used by the mobile app.

**Data Pipeline**:
```
hymnal.net  → crawler (Python) ─┐
                                ├→ dedup/merge → hymns/*.json → Flutter app → User
songbase.life → crawler (API) ──┘
```

**Data Source**: Hymnal data (songs, lyrics, chords) is loaded from local JSON assets bundled with the app. The JSON files are generated by the crawler. The app uses an Isar database for search functionality and metadata indexing.

## Full Build Pipeline (Crawl → App)

The easiest way to rebuild all hymn data and prepare the app:

```bash
# Full pipeline: crawl everything, copy to app, regenerate manifest
./build_hymns.sh

# Only re-crawl songbase.life (fast, ~3 seconds via API)
./build_hymns.sh --songbase-only

# Skip crawling, just copy existing crawler output and rebuild manifest
./build_hymns.sh --skip-crawl
```

**What the pipeline does:**
```
1. cd crawler && python3 crawl_all.py    # Crawl hymnal.net + songbase.life
   - Phase 1-2: Crawl hymnal.net (ch, ts, h, ns, lb, nt)
   - Phase 3: Crawl songbase.life API → crawler/hymns_songbase/
   - Phase 4: Deduplicate & merge songbase into crawler/hymns/
   - Phase 5: Convert Chinese to simplified
   - Phase 6: Apply manual edits from crawler/hymns_manual/
2. cp crawler/hymns/*.json hymns/        # Copy to app asset directory
3. dart run tool/build_database.dart     # Regenerate assets/available_hymns.json
4. flutter run                           # Run the app
```

**Manual steps (if not using the script):**
```bash
# 1. Crawl
cd crawler
python3 crawl_all.py

# 2. Copy output to app directory
cd ..
mkdir -p hymns
cp crawler/hymns/*.json hymns/

# 3. Rebuild the hymn manifest
dart run tool/build_database.dart

# 4. Run the app
flutter run
```

## Data Structure

### Hymn JSON Files (assets/hymns/)

Hymn files are named with the format: `{bookId}_{number}.json` (e.g., `ts_1.json`, `ch_100.json`, `h_500.json`, `ns_50.json`)

**Key Terminology**:
- **bookId**: The identifier for the hymn book (e.g., "ts", "ch", "h", "ns"). This is part of the hymn ID.
- **category**: The descriptive category from metadata (e.g., "Ultimate Manifestation", "God's Economy"). This is NOT the same as bookId.
- **hymnId**: The full identifier combining bookId and number (e.g., "ts_1", "ch_100")

**Book IDs**:
- `ts`: 補充本 (Supplement) - Chinese
- `ch`: 大本 (Classic Chinese Hymns) - Chinese
- `h`: Hymns (English)
- `ns`: New Songs (English/Chinese mix)
- `lb`: New Songs lb variant (English)
- `nt`: New Tune (English)
- `sb`: SongBase (English, from songbase.life — uses songbase song ID as number)

**JSON Structure**:
```json
{
  "title": "Hymn Title",
  "url": "https://www.hymnal.net/...",
  "verses": [
    {
      "type": "verse",
      "lines": [
        {
          "segments": [
            {"text": "Amazing grace", "chord": {"name": "G"}},
            {"text": " how sweet the sound", "chord": {"name": "C"}}
          ]
        }
      ]
    }
  ],
  "metadata": {
    "category": "Ultimate Manifestation",
    "time": "3/4",
    "hymn_code": "51123321271",
    "related": [
      {
        "url": "https://www.hymnal.net/en/hymn/ch/100",
        "language": "zh",
        "category": "ch",
        "number": "100",
        "title": "Ch100"
      }
    ]
  },
  "alternate_versions": [
    {
      "source": "songbase",
      "title": "Alternate Title",
      "url": "https://songbase.life/song/123",
      "verses": [...],
      "metadata": {"source": "songbase", "songbase_id": 123, "capo": 3}
    }
  ]
}
```

### Database Models

**HymnDb** (Isar database model for search):
- `hymnId`: Full hymn identifier (e.g., "ts_1")
- `bookId`: Book identifier extracted from hymnId (e.g., "ts")
- `title`: Hymn title
- `number`: Hymn number within the book
- `category`: Descriptive category from metadata (e.g., "Ultimate Manifestation")
- `fullText`: Concatenated text for full-text search
- `hasAlternateVersions`: Whether this hymn has alternate versions from other sources
- Indexed fields: hymnId, bookId, title, fullText (case-insensitive)

**HymnSong** (Display model):
- `title`: Hymn title
- `url`: Source URL
- `verses`: List of Verse objects containing Lines and Segments
- `alternateVersions`: Optional list of HymnVersion objects from other sources

**HymnVersion** (Alternate version from another source):
- `source`: Source identifier (e.g., "songbase")
- `title`: Title from this source
- `url`: URL on the source site
- `verses`: List of Verse objects
- `metadata`: Map containing category, time, hymn_code, related hymns, etc.

### Important Notes

- **Always use `bookId`** when navigating or loading hymns (not `category`)
- **category** is a descriptive field from metadata and may contain values like "Ultimate Manifestation", "God's Economy", etc.
- **bookId** identifies which hymnal book the song comes from ("ts", "ch", "h", "ns")
- The hymn files use `"category"` in the related hymns section to mean `bookId` - this is a JSON structure naming convention

## Songbase Deduplication and Merging

The app merges hymns from two sources: hymnal.net (primary) and songbase.life.

### How it works

**`h` (English Hymns):** hymnal.net and songbase's `english_hymnal` share the same numbering. Dedup is done by filename — `h_123.json` from both sources refer to the same hymn. If lyrics differ, the songbase version is added as an `alternate_versions` entry. ~1,257 out of 1,348 songbase h songs have different lyrics from hymnal.net.

**`sb` (SongBase):** All English songs on songbase.life that are NOT in `english_hymnal` get saved as `sb_{songbase_id}.json`, using songbase's own song ID as the number. This includes songs from songbase's "Blue Songbook" and bookless songs. These are kept separate — no dedup against `ns` or other hymnal.net books.

**`ns` (New Songs):** Kept as-is from hymnal.net, no songbase merging.

### Songbase crawler mapping (`songbase_crawler/crawler.py`)

- `english_hymnal` → `h_*.json` — same numbering, deduped by filename
- All other English songs → `sb_{songbase_id}.json` — no dedup, separate catalog

### build_database.dart

The `sb` category uses directory listing instead of range scanning (songbase IDs are sparse, up to ~12000+). Other categories use range scan 1–9999.

## Common Issues

**Database not rebuilding**: Ensure you increment `_currentDbVersion` in `hymn_db_service.dart` after schema changes, then run the app. The database will automatically repopulate.

**Build runner errors**: Run `dart run build_runner build --delete-conflicting-outputs` after modifying any Isar collection models (files with `@collection` annotation).

**Navigation errors**: Always use `bookId` (not `category`) when navigating to hymns. The `category` field is descriptive metadata, while `bookId` identifies the hymnal book ("ts", "ch", "h", "ns").

**Favorites not persisting**: Favorites are stored in SharedPreferences with key `'favorite_hymns'` as a list of hymnIds. Check that SharedPreferences is working correctly on the target platform.
